# 脉诊数据分析模块开发计划文档

## 1. 项目定位

本模块的核心目标不是直接进行“脉象诊断”或“证候分类”，而是首先建立脉诊数据的可信度评估体系。当前数据来自 20 个用户，每个用户在早、中、晚等不严格时间点进行多次脉诊采集，存在缺失、时间不规则、设备混用和佩戴位置敏感等问题。因此，模块开发的第一原则是先判断采集数据是否可靠，再分析设备差异、重复测量一致性、日内变化、个体稳定性和多模态相关性。

模块的分析主线为：

采集质量评估 → 单次测量漂移检测 → 可靠片段定位 → 去漂移补偿评估 → 设备一致性分析 → 特征可靠性分级 → 重复测量可靠性分析 → 日内变化分析 → 个体稳定性分析 → 多模态相关分析。

其中，前六项属于“数据可信度层”，后四项属于“生理与中医表征分析层”。在数据可信度层未建立之前，后续关于早中晚差异、个体脉象特征、多模态相关性的结果只能作为探索性发现，不能作为正式结论。

## 2. 开发优先级总览

### P0：单次测量漂移分析

单次测量内的信号漂移是目前影响最大的误差来源。传感器佩戴位置、接触压力、手腕姿态变化和一次测量中的轻微移动，都可能导致波形基线、幅值、形态和周期稳定性发生变化。因此，系统必须首先识别一次测量内部是否存在漂移，并判断漂移发生在哪些时间段。

核心输出包括：

- drift_severity_index：漂移严重度指数
- baseline_drift_slope：基线漂移斜率
- amplitude_drift_slope：主波幅值漂移斜率
- morphology_drift_score：波形形态漂移分数
- cycle_instability_score：搏动周期不稳定分数
- change_point_count：变点数量
- max_change_magnitude：最大突变幅度
- drift_start_time：漂移开始时间
- drift_segments：漂移片段范围

### P1：可靠片段定位

漂移检测用于发现问题，可靠片段定位用于决定最终使用哪一段数据。一次测量不应默认取全程平均，而应先切分为多个滑动窗口，再基于窗口质量评分寻找最稳定、最可靠的连续片段。

核心输出包括：

- window_quality_score：窗口质量评分
- best_segment_start_time：推荐片段开始时间
- best_segment_end_time：推荐片段结束时间
- best_segment_duration：推荐片段时长
- stable_segment_ratio：稳定片段占总时长比例
- valid_beat_ratio：有效搏动比例
- invalid_segment_reason：无效片段原因
- measurement_validity_label：测量有效性标签，包括有效、局部有效、无效

### P2：去漂移补偿模型

去漂移补偿不是为了强行修复低质量数据，而是为了修正可解释、可建模的非生理性偏移。对于基线缓慢漂移、幅值整体缩放、接触压力变化导致的平滑趋势，可以尝试补偿。对于波形形态突变、峰谷结构明显改变、窗口间形态差异过大的数据，应优先切除或判定为无效，而不是补偿。

核心输出包括：

- compensated_waveform：补偿后波形
- compensation_method：补偿方法
- compensation_residual：补偿残差
- baseline_stability_improvement：基线稳定性改善率
- amplitude_cv_improvement：幅值变异系数改善率
- morphology_consistency_improvement：形态一致性改善率
- compensation_validity_label：补偿有效性标签
- use_compensated_or_raw：后续分析使用原始数据、补偿数据或仅使用可靠片段

### P3：单次测量质量评分

单次测量质量评分是系统级入口指标，用于快速判断一次采集是否可以进入后续分析。评分应可解释，不能只输出一个黑箱分数。建议将总评分拆分为接触质量、噪声水平、基线稳定性、幅值稳定性、周期稳定性和形态一致性几个维度。

核心输出包括：

- signal_quality_score：信号质量总评分
- contact_quality_score：接触质量评分
- noise_score：噪声评分
- baseline_stability_score：基线稳定性评分
- amplitude_stability_score：幅值稳定性评分
- cycle_stability_score：周期稳定性评分
- morphology_consistency_score：形态一致性评分
- peak_detection_success_rate：峰谷识别成功率
- artifact_ratio：伪影比例

### P4：两台设备一致性分析

设备差异是第二大误差来源。两台设备不能简单混合分析，必须先判断每个特征是否存在系统偏差、随机误差、个体交互效应和可校正性。设备分析应只基于高质量稳定片段进行，不能直接使用未经质量控制的全量数据。

核心输出包括：

- device_bias：设备间平均偏差
- device_relative_bias：设备间相对偏差
- device_icc：跨设备组内相关系数
- device_cv：跨设备变异系数
- bland_altman_mean_diff：Bland-Altman 平均差
- bland_altman_loa_upper：一致性上限
- bland_altman_loa_lower：一致性下限
- device_subject_interaction：设备与用户交互效应
- device_time_interaction：设备与时间段交互效应
- device_correction_model：设备校正模型
- device_merge_policy：设备数据合并策略

设备数据合并策略分为四类：

- A 类：跨设备可直接合并
- B 类：需设备校正后合并
- C 类：只能在同设备内比较
- D 类：设备差异过大，暂不推荐用于核心分析

### P5：特征可靠性分级

不同脉象特征对漂移、佩戴位置和设备差异的敏感性不同。系统应对每个特征进行可靠性分级，以决定其是否能进入后续主分析。

核心输出包括：

- drift_sensitivity：漂移敏感性
- device_sensitivity：设备敏感性
- within_session_cv：单次测量内变异系数
- within_session_icc：单次测量内一致性
- repeatability_icc：近时间重复测量一致性
- missing_rate：缺失率
- outlier_rate：异常值比例
- quality_dependency_score：对信号质量的依赖程度
- feature_reliability_grade：特征可靠性等级

特征可靠性等级建议分为四级：

- A 级：高可靠特征，可进入所有核心分析
- B 级：中等可靠特征，可进入主分析，但需控制设备和质量因素
- C 级：探索性特征，仅用于辅助分析
- D 级：不可靠特征，暂不进入正式分析

### P6：近时间重复测量可靠性分析

在完成漂移控制、可靠片段定位、设备控制和特征可靠性分级之后，再分析同一个人在 30 分钟内或相近时间多次采集的可靠性。该分析用于判断脉诊特征在短时间内是否稳定，以及单次变化超过多少才可能是真实变化。

核心输出包括：

- within_subject_near_time_icc：同一人近时间 ICC
- same_device_repeatability：同设备重复性
- cross_device_repeatability：跨设备重复性
- sem：标准误差
- mdc：最小可检测变化
- repeat_measurement_cv：重复测量 CV
- repeat_absolute_error：重复测量绝对误差
- repeat_relative_error：重复测量相对误差

### P7：日内变化分析

日内变化分析用于比较同一用户早、中、晚脉象特征的差异。由于当前采集时间不严格，系统不应只使用固定早中晚标签，还应记录连续时间变量，例如采集小时数、距离上次进食时间、距离运动时间等。

核心输出包括：

- time_of_day_label：早、中、晚标签
- collection_hour：采集小时
- morning_noon_evening_difference：早中晚差异
- subject_centered_time_effect：个体中心化后的时间效应
- mixed_model_time_effect：混合效应模型中的时间段效应
- diurnal_variation_magnitude：日内波动幅度
- diurnal_variation_consistency：日内变化方向一致性
- exceed_mdc_flag：日内变化是否超过最小可检测变化

日内变化只有在超过测量误差、设备误差和特征自身波动范围时，才可被认为可能具有生理意义。

### P8：个体内一致性与个体间差异分析

该分析用于判断某些脉象特征更像“个体稳定表型”，还是更像“短期状态指标”。如果某一特征个体间方差明显大于个体内方差，则更适合作为长期体质或个体特异性指标。如果个体内波动更大，则更适合作为状态监测指标。

核心输出包括：

- within_subject_distance：个体内距离
- between_subject_distance：个体间距离
- between_within_ratio：个体间/个体内距离比
- subject_variance_ratio：个体间方差占比
- day_variance_ratio：日期方差占比
- time_period_variance_ratio：同日时间段方差占比
- device_variance_ratio：设备方差占比
- residual_variance_ratio：残差方差占比
- subject_separability_score：个体可分性分数

### P9：脉象与舌象、面象等多模态相关分析

多模态相关分析应放在较后阶段进行，因为它最容易受到设备差异、漂移、时间段和个体差异的污染。分析时必须区分个体间相关和个体内相关。

核心输出包括：

- pulse_tongue_between_subject_correlation：脉象与舌象的个体间相关
- pulse_tongue_within_subject_correlation：脉象与舌象的个体内相关
- pulse_face_between_subject_correlation：脉象与面象的个体间相关
- pulse_face_within_subject_correlation：脉象与面象的个体内相关
- repeated_measure_correlation：重复测量相关
- partial_correlation_controlled：控制设备、时间段和质量后的偏相关
- multimodal_sync_index：多模态同步变化指数
- multimodal_feature_network：多模态特征相关网络

个体间相关回答“哪些人长期具有相似的脉象、舌象和面象特征”；个体内相关回答“同一个人在状态变化时，不同模态是否同步变化”。二者应分开报告，不能混合解释。

## 3. 数据结构设计

### 3.1 用户表 user

字段建议：

- user_id：用户唯一编号
- sex：性别
- age：年龄
- height：身高
- weight：体重
- baseline_health_status：基础健康状态
- chronic_disease_history：慢性病史
- medication_status：用药情况
- constitution_label：已有体质标签，如有
- notes：备注

### 3.2 设备表 device

字段建议：

- device_id：设备唯一编号
- device_model：设备型号
- sensor_type：传感器类型
- firmware_version：固件版本
- calibration_date：最近校准时间
- sampling_rate：采样率
- pressure_sensor_available：是否有压力传感器
- imu_available：是否有姿态传感器
- device_notes：设备备注

### 3.3 测量表 measurement

一次完整脉诊采集对应一条 measurement 记录。

字段建议：

- measurement_id：测量唯一编号
- user_id：用户编号
- device_id：设备编号
- start_time：采集开始时间
- end_time：采集结束时间
- duration：采集时长
- time_of_day_label：早、中、晚或其他时间标签
- collection_hour：采集小时
- hand_side：左手或右手
- pulse_position：寸、关、尺或其他采集位置
- posture：采集姿态
- operator_id：操作者编号
- raw_waveform_path：原始波形存储路径
- pressure_signal_path：压力信号路径，如有
- imu_signal_path：姿态信号路径，如有
- user_feedback：使用者反馈
- measurement_notes：测量备注

### 3.4 窗口表 window

一次 measurement 被切分为多个滑动窗口，每个窗口对应一条 window 记录。

字段建议：

- window_id：窗口唯一编号
- measurement_id：所属测量编号
- start_offset：窗口相对起始时间
- end_offset：窗口相对结束时间
- window_duration：窗口时长
- beat_count：窗口内搏动数量
- valid_beat_count：有效搏动数量
- window_quality_score：窗口质量评分
- baseline_level：基线水平
- baseline_slope：基线斜率
- amplitude_mean：主波幅值均值
- amplitude_cv：主波幅值变异系数
- cycle_mean：搏动周期均值
- cycle_cv：搏动周期变异系数
- morphology_similarity：窗口内波形形态一致性
- template_similarity_to_global：与全局模板的相似度
- artifact_ratio：伪影比例
- window_validity_label：窗口有效性标签

### 3.5 搏动表 beat

每个脉搏周期对应一条 beat 记录。

字段建议：

- beat_id：搏动唯一编号
- measurement_id：所属测量编号
- window_id：所属窗口编号
- beat_start_time：搏动起点
- beat_end_time：搏动终点
- cycle_length：搏动周期长度
- main_peak_time：主波峰时间
- main_peak_amplitude：主波幅值
- trough_time：谷值时间
- dicrotic_wave_time：重搏波时间
- dicrotic_wave_amplitude：重搏波幅值
- beat_area：单搏面积
- rise_time：上升时间
- decay_time：下降时间
- beat_quality_score：单搏质量评分
- beat_validity_label：搏动有效性标签

### 3.6 特征表 pulse_feature

每次 measurement 或推荐可靠片段生成一组特征。

字段建议：

- feature_id：特征记录编号
- measurement_id：测量编号
- source_segment_type：全段、推荐片段、补偿后片段
- segment_start_time：特征来源片段开始时间
- segment_end_time：特征来源片段结束时间
- feature_name：特征名称
- feature_value：特征值
- feature_unit：单位
- feature_category：时域、频域、形态学、非线性、中医标签等
- quality_weight：该特征的质量权重
- reliability_grade：特征可靠性等级

## 4. 算法模块设计

### 4.1 原始波形预处理模块

功能目标：

- 读取原始脉搏波数据
- 去除明显异常值
- 基线初步校正
- 滤波去噪
- 标准化采样率
- 保留原始波形与预处理波形两个版本

处理原则：

- 原始数据不可覆盖
- 所有预处理操作需要记录参数
- 不同处理版本需要可追溯
- 后续漂移分析应同时参考原始波形和预处理波形

### 4.2 搏动周期检测模块

功能目标：

- 自动识别每个搏动周期的起点、峰值、谷值
- 标记异常搏动
- 计算每搏质量评分
- 为窗口级特征计算提供基础

核心判断：

- 峰值是否清晰
- 周期长度是否合理
- 峰谷结构是否完整
- 是否存在尖峰噪声
- 是否存在平线或饱和
- 相邻搏动形态是否异常突变

### 4.3 滑动窗口质量评估模块

建议默认参数：

- 窗口长度：10 秒或 15 秒
- 步长：2 秒或 5 秒
- 每个窗口至少包含 6 至 10 个有效搏动
- 窗口质量评分范围：0 至 100

窗口质量评分由以下维度组成：

- 峰谷识别成功率
- 有效搏动比例
- 信噪比
- 基线稳定性
- 幅值稳定性
- 周期稳定性
- 波形形态一致性
- 伪影比例

### 4.4 漂移检测模块

漂移检测应分为趋势型漂移和突变型漂移。

趋势型漂移包括：

- 基线缓慢上移或下移
- 主波幅值缓慢升高或降低
- 波形面积持续变化
- 窗口形态相似度逐渐下降

突变型漂移包括：

- 某个时间点后主波幅值突然变化
- 基线突然跳变
- 波形形态突然改变
- 峰谷识别模式突然失效

输出结果：

- 是否存在漂移
- 漂移类型
- 漂移严重度
- 漂移发生时间
- 漂移影响片段
- 是否可补偿
- 是否应切除

### 4.5 可靠片段定位模块

可靠片段定位逻辑：

- 首先排除低质量窗口
- 再排除漂移严重窗口
- 从剩余窗口中寻找最长连续高质量片段
- 若存在多个候选片段，优先选择质量评分最高、窗口内变异最小、形态最稳定的片段
- 若候选片段之间特征差异较大，不应合并
- 若无连续足够长度的高质量片段，则该次测量判为无效

建议判定规则：

- 推荐片段长度 ≥ 20 秒，且质量评分 ≥ 80：有效
- 推荐片段长度 10 至 20 秒，且质量评分 ≥ 70：局部有效
- 推荐片段长度 < 10 秒，或质量评分 < 70：无效
- 发生严重形态突变且无法定位稳定片段：无效
- 补偿后质量无明显改善：优先使用原始可靠片段，而非补偿全段

### 4.6 去漂移补偿模块

可尝试补偿的情况：

- 基线平滑漂移
- 幅值整体缩放
- 低频趋势干扰
- 接触压力缓慢变化导致的线性或非线性偏移

不建议补偿的情况：

- 波形形态突变
- 主波、潮波、重搏波结构关系改变
- 峰谷识别不稳定
- 伪影比例过高
- 窗口间模板相似度明显下降
- 补偿后重复性没有改善

补偿模型评估原则：

- 补偿后同次重复性应提高
- 补偿后跨设备一致性应提高
- 补偿后个体差异不应被过度抹平
- 补偿后残差应可解释
- 补偿前后所有关键特征变化应被记录

### 4.7 设备一致性分析模块

分析前提：

- 只使用高质量稳定片段
- 只比较近时间采集数据
- 优先使用同一用户、同一时间窗口、两台设备配对测量数据
- 如存在采集顺序差异，应记录设备采集顺序

分析方法：

- 配对均值差分析
- 相对偏差分析
- Bland-Altman 一致性分析
- ICC 分析
- 混合效应模型
- 设备校正模型
- 设备 × 用户交互分析
- 设备 × 时间段交互分析

最终输出：

- 每个特征的设备一致性等级
- 每个特征是否允许跨设备合并
- 是否需要设备校正
- 校正后残差是否可接受
- 设备差异是否大于真实生理变化

### 4.8 特征可靠性分级模块

特征可靠性综合评分由以下部分组成：

- 单次测量内稳定性
- 近时间重复测量稳定性
- 跨设备一致性
- 对漂移的敏感性
- 对信号质量的依赖性
- 缺失率
- 异常值比例
- 是否可解释
- 是否经过补偿后稳定性提高

最终输出每个特征的可靠性等级。

### 4.9 日内变化分析模块

分析原则：

- 必须使用 A 级或 B 级特征
- 必须使用高质量片段
- 必须控制设备因素
- 必须判断日内变化是否超过 MDC
- 不应只比较原始均值，应优先使用个体中心化后的变化

分析方法：

- 个体内早中晚差异
- 混合效应模型
- 时间连续变量建模
- 日内变化方向一致性分析
- 超过测量误差阈值的比例分析

### 4.10 个体稳定性分析模块

分析原则：

- 区分个体稳定表型与短期状态波动
- 区分个体间方差、个体内日期方差、同日时间段方差、设备方差和残差
- 不应把设备差异误认为个体差异

分析方法：

- 方差成分分解
- 个体内距离与个体间距离比较
- between/within ratio
- 聚类可分性分析
- 身份可分性探索分析

### 4.11 多模态相关分析模块

分析对象：

- 脉象特征
- 舌象特征
- 面象特征
- 问诊指标
- 睡眠、运动、饮食、情绪等状态变量，如有

分析原则：

- 区分个体间相关和个体内相关
- 控制设备因素
- 控制时间段因素
- 控制信号质量因素
- 不以小样本探索结果作为强结论

分析方法：

- 个体间相关
- 个体内相关
- repeated-measures correlation
- 偏相关
- 个体中心化相关
- 多模态特征网络
- PLS 或 CCA 探索性分析

- 同设备重复性
- 跨设备一致性
- 设备系统偏差
- 佩戴位置敏感性
- 姿态移动导致的漂移模式

## 12. 风险与控制策略

### 风险一：样本量较小

当前只有 20 个用户，不适合训练复杂预测模型。应优先做可靠性、稳定性和探索性统计分析。

控制策略：

- 优先使用描述性分析和混合效应模型
- 结果分为探索性和验证性
- 不做过度泛化
- 后续逐步扩大样本量

### 风险二：时间点不规则

早中晚采集不是严格实验设计，可能导致时间段分析偏差。

控制策略：

- 同时保留离散时间标签和连续时间变量
- 使用混合效应模型
- 控制用户、设备和日期效应
- 判断变化是否超过 MDC

### 风险三：设备差异污染结论

两台设备可能存在系统偏差和用户交互效应。

控制策略：

- 所有分析记录 device_id
- 先做设备一致性分析
- 特征级决定是否合并设备
- 必要时进行设备校正

### 风险四：佩戴位置敏感

传感器位置和接触压力可能显著影响波形。

控制策略：

- 单次测量内进行漂移检测
- 使用可靠片段而非全段平均
- 增加佩戴扰动验证实验
- 建立特征漂移敏感性分级

### 风险五：多模态伪相关

脉象、舌象、面象之间可能因为个体差异产生表面相关。

控制策略：

- 区分个体间相关和个体内相关
- 控制设备、时间段和质量因素
- 使用个体中心化分析
- 多模态结论先作为探索性结果

## 14. 总结

本模块开发应坚持一个基本原则：先解决数据可信度，再解释脉象变化。当前最优先开发的不是中医诊断模型，而是单次测量漂移分析、可靠片段定位、去漂移补偿评估、设备一致性分析和特征可靠性分级。

只有当系统能够判断“这次测量是否可靠”“哪一段波形最可靠”“两台设备是否可比”“哪些特征稳定可用”之后，早中晚差异、个体稳定性和脉象—舌象—面象相关性分析才具有实际意义。

因此，第一版开发重点应明确聚焦于：

单次测量漂移分析 + 可靠片段定位 + 去漂移补偿评估 + 设备一致性分析 + 特征可靠性分级。

这五个模块构成脉诊数据分析系统的可信度底座。
