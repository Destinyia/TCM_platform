# 开发计划补充：结构化宽表

更新时间：2026-05-20

## 当前阶段

目标：在现有“标准化入库 + 结构化解析”的基础上，新增 visit 级宽表，使多源解析结果能以“纵轴为 visit、横轴为变量”的二维结构展示、查询和导出。

## 已纳入本阶段的工作

1. 新增数据库表 `mart_visit_feature_wide`。
2. 新增宽表构建服务 `backend/app/feature_wide.py`。
3. `POST /api/ingest/parse-structured-data` 在完成模态解析后自动重建宽表。
4. `GET /api/demo/visits/{visit_id}` 返回 `feature_wide`。
5. visit 详情抽屉新增“结构化宽表字段”，按模态折叠/展开显示。

## 第一阶段边界

- 先抽取玉生堂 JSON 中稳定标量字段。
- 大段脉搏波形不直接进入宽表，只保留摘要。
- 中科 Excel 暂以解析资产和匹配行追溯为主，后续补专用解析。
- 宽表物理结构采用“固定元数据列 + JSONB 动态变量”，等变量字典稳定后再固化高频列。

## 下一步

1. 为中科 Excel 增加专用解析器，按 sheet 类型输出结构化记录。
2. 新增明细表：
   - `fact_ask_answer`
   - `fact_pulse_position_feature`
   - `fact_tongue_region_color`
   - `fact_voice_item_feature`
   - `fact_pulse_waveform_asset`
3. 增加 `dim_feature_variable` 维护变量字典。
4. 增加宽表下载接口，支持 CSV/Parquet。

## 2026-05-20 进度更新：脉诊解析与展示

本次已完成脉诊数据的第一版结构化解析和前端呈现闭环。

已完成：

1. 新增 `backend/app/pulse_parser.py`，将脉诊解析从通用 JSON/Excel 摘要中拆出为独立模块。
2. 玉生堂 `dataPulse.json` 已解析为标准 `records[]`：
   - `pulse_rate`
   - `force`
   - `tension`
   - `fluency`
   - `amplitude`
   - `stability_score`
   - `waveform_summary`
   - `waveform_preview`
3. 玉生堂波形预览来自 `SinglePluse`、`Cun`、`GuanMai`、`Chi` 等波形向量的降采样结果，不使用缓存图片。
4. 中科 `脉诊数据*.xls` 已按 visit 病例号匹配行块，解析整体脉象、六部位测量明细和聚合指标。
5. `parsed_structured_data_json.records` 成为脉诊分析页、visit 详情和后续宽表抽取的统一数据来源。
6. `visit_feature_wide` 升级为 `visit_feature_wide_v2`，脉诊特征优先从 `records[]` 抽取。
7. `PulseAnalysis.vue` 已改为基于结构化 records 展示横向、纵向、同槽位稳定性、不同时段变化和当前记录详情。
8. `VisitDetailDrawer.vue` 已在脉诊模态下展示解析后的 records 表格，并保留原始资产展示。

下一步：

1. 将中科六部位测量明细固化为独立明细表，例如 `fact_pulse_position_feature`。
2. 将玉生堂完整波形向量迁移为独立波形资产或 Parquet/NPY 文件索引，宽表继续只保留摘要与预览。
3. 为脉诊变量建立 `dim_feature_variable` 字典，补中文名、来源、单位、取值范围和是否可进入训练集。
4. 增加脉诊 records 的导出接口，用于机器学习样本构建和 benchmark。
