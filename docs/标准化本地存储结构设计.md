# 标准化本地存储结构设计

生成时间：2026-05-16

本文定义一阶段及后续本地文件系统的标准化存储结构。目标是在不启用自动化 ETL 管线的情况下，先用本地静态 mock 数据和人工整理数据构造可追溯、可展示、可导出、可迁移到对象存储的文件组织方式。

## 1. 设计目标

1. 原始文件不可覆盖、不可重命名作为唯一证据。
2. 标准资产统一编号、统一 URI、统一 hash 管理。
3. 前端展示、脉诊分析、数据集下载都只读标准资产和数据库元数据。
4. 训练集、benchmark、导出物必须有版本目录和 manifest。
5. 本地路径必须可迁移到 MinIO/S3，不能把 Windows 绝对路径写死到业务逻辑。

## 2. 根目录配置

推荐使用环境变量：

```env
TCM_STORAGE_ROOT=E:\workspace\TCM_platform\storage
TCM_STORAGE_URI_PREFIX=local://tcm-platform
```

WSL 访问同一目录时，对应路径通常为：

```bash
/mnt/e/workspace/TCM_platform/storage
```

数据库中不要保存仅 Windows 可用的绝对路径作为主要访问地址。建议保存：

- `storage_uri`：平台统一 URI，例如 `local://tcm-platform/assets/...`
- `source_relative_path`：相对批次根目录的原始路径
- `file_path`：审计用原始绝对路径，可选，仅限高权限查看
- `file_hash`：文件内容 hash
- `file_size`
- `mime_type`

## 4. 各目录职责

### 4.1 `landing/`

人工放入待处理文件，不作为长期证据。

| 目录 | 用途 |
| --- | --- |
| `incoming/` | 新收到但未登记的数据包 |
| `staged/` | 已确认来源和批次、准备登记的数据包 |
| `rejected/` | 明确不入库的数据包 |

规则：

- `landing` 内文件可以移动。
- 一旦进入 `raw`，不再修改目录内容。

### 4.2 `raw/`

原始证据区，按来源和批次保留。

推荐结构：

```text
raw/{source_vendor}/{batch_id}/
  manifest.json
  source/
    原始目录或文件
  inventory/
    file_inventory.csv
    file_inventory.jsonl
    parse_probe.jsonl
```

`manifest.json` 建议字段：

```json
{
  "batch_id": "demo_20260516_001",
  "source_vendor": "zhongke",
  "source_batch_name": "2026.04.1~15",
  "received_at": "2026-05-16T00:00:00+08:00",
  "operator": "manual",
  "root_policy": "immutable_after_registered",
  "notes": "phase1 local demo"
}
```

规则：

- `source/` 内保留原始目录结构。
- 原始文件不重命名、不覆盖。
- 如果同一批次重复接收，生成新的 `batch_id`，不要覆盖旧批次。

### 4.3 `standard/records/`

保存标准化来源记录的 JSON 快照，便于前端 demo 和 API 调试。

推荐结构：

```text
standard/records/{source_vendor}/{source_record_group_id}/
  record.json
  modality_index.json
```

`record.json` 保存来源聚合证据：

```json
{
  "source_vendor": "zhongke",
  "source_visit_id": "ZK-20260415001",
  "source_record_group_id": "zhongke-ZK-20260415001-U001-0802",
  "canonical_name": "爱丽娜",
  "collected_at": "2026-04-15T08:02:00+08:00",
  "quality_flags": ["source_caseid_time_cluster_5m"],
  "source_evidence": []
}
```

`modality_index.json` 保存该记录下每个模态的资产和解析状态。

### 4.4 `standard/assets/`

标准资产区，只保存平台可引用资产。

推荐路径：

```text
standard/assets/{asset_type}/{hash_prefix}/{asset_id}.{ext}
```

示例：

```text
standard/assets/tongue_origin/ab/8f2e3c9a-...-origin.jpg
standard/assets/pulse_raw_left/02/7a12b8d4-...-left.bin
standard/assets/report_pdf/71/9c33a1e2-...-report.pdf
standard/assets/voice_wav/e4/3f4b8a1c-...-a.wav
```

命名规则：

- `asset_id` 使用 UUID。
- `hash_prefix` 使用 `sha256` 前 2 位，避免单目录文件过多。
- 扩展名来自解析后的真实类型，不完全相信原始扩展名。
- 无扩展名脉波文件统一使用 `.bin`，并在数据库中记录 `asset_type=pulse_raw_left/right`。

数据库映射：

| 字段 | 示例 |
| --- | --- |
| `asset_id` | `8f2e3c9a-...` |
| `asset_type` | `tongue_origin` |
| `asset_role` | `diagnostic_image` |
| `file_name` | 原始文件名 |
| `file_path` | 原始绝对路径，高权限审计用 |
| `storage_uri` | `local://tcm-platform/standard/assets/tongue_origin/ab/8f2e3c9a.jpg` |
| `file_hash` | `sha256:ab...` |
| `mime_type` | `image/jpeg` |

### 4.5 `standard/thumbnails/` 与 `standard/previews/`

用于前端快速展示。

| 目录 | 用途 |
| --- | --- |
| `thumbnails/` | 舌图、面图、PDF 首页缩略图 |
| `previews/pdf/` | PDF 转图或 OCR 摘要 |
| `previews/audio/` | 音频短预览或转码文件 |
| `previews/waveform/` | 脉波、语音波形图 |

规则：

- 预览文件是派生物，必须能从原始资产重建。
- 预览文件也应登记 `fact_file_asset` 或派生资产表，`asset_role=preview`。

### 4.6 `feature/`

保存分析用派生特征，不保存原始大文件副本。

推荐格式：

- 小规模 demo：JSONL/CSV
- 中后期：Parquet
- 波形数组：`.npy`、`.npz` 或 Parquet list column

示例：

```text
feature/pulse/summary/pulse_summary_v20260516.parquet
feature/pulse/raw_signal/{asset_id}.npz
feature/voice/acoustic/voice_acoustic_v20260516.parquet
```

### 4.7 `mart/`

保存展示和研究用宽表快照。

示例：

```text
mart/user_day_panel/user_day_panel_v20260516.parquet
mart/quality_summary/quality_summary_v20260516.csv
```

规则：

- mart 是快照，不覆盖历史版本。
- 版本号应包含规则版本或生成日期。

### 4.8 `datasets/`

保存可下载标准化数据集。

推荐结构：

```text
datasets/DS-PULSE-001/v2026.05.demo.001/
  dataset_card.md
  manifest.jsonl
  metadata.csv
  splits/
    train.jsonl
    val.jsonl
    test.jsonl
  files/
    pulse/
      {sample_id}_{asset_id}.bin
  checksums.sha256
```

`manifest.jsonl` 每行一个样本：

```json
{
  "sample_id": "demo-sample-001",
  "user_id": "DEID-001",
  "visit_id": "V001",
  "split": "train",
  "modalities": ["pulse"],
  "assets": [
    {
      "asset_id": "A002",
      "asset_type": "pulse_raw_left",
      "path": "files/pulse/demo-sample-001_A002.bin",
      "sha256": "..."
    }
  ],
  "features": {
    "pulse_rate": 72,
    "force": 62
  },
  "label": {
    "pulse_type": "弦细"
  },
  "quality_flags": []
}
```

规则：

- 数据集目录不可原地修改。
- 新筛选条件、新标签版本、新 split seed 都生成新版本目录。
- 默认按用户划分 train/val/test，避免同一用户跨集合泄漏。

### 4.9 `labels/`

保存标签体系和标注版本。

```text
labels/pulse_type/v2026.05.demo.001/
  label_schema.json
  annotations.jsonl
  review_log.jsonl
```

规则：

- `label_schema.json` 定义标签字典。
- `annotations.jsonl` 保存标签值、来源、标注人、审核状态。
- 标签版本进入 `dataset_version`，保证数据集可复现。

### 4.10 `benchmarks/`

保存 benchmark 输入输出与报告。

```text
benchmarks/BM-PULSE-001/run_20260516_001/
  prediction.jsonl
  metrics.json
  report.md
  artifacts/
```

规则：

- 模型文件可以放 `artifacts/`，但首期也可以只放预测结果和指标。
- 正式接 MLflow 后，本目录保存平台侧可下载副本或报告摘要。

### 4.11 `quarantine/`

隔离不能正常进入标准资产的数据。

| 目录 | 用途 |
| --- | --- |
| `parse_failed/` | 解析失败 |
| `duplicate_export/` | 重复导出副本 |
| `pii_risk/` | 疑似隐私泄露风险 |
| `unknown_type/` | 文件类型未知 |

规则：

- 隔离不是删除。
- 隔离文件仍应有审计记录。
- 后续人工确认后可重新登记到 `standard/assets/`。

### 4.12 `logs/` 与 `tmp/`

`logs/` 保存运行日志，`tmp/` 保存临时文件。

规则：

- `tmp/` 可以清理。
- `logs/` 按日期滚动，不进入数据集。

## 5. URI 规范

本地 URI 统一使用：

```text
local://tcm-platform/{relative_path}
```

示例：

```text
local://tcm-platform/raw/zhongke/demo_20260516_001/source/...
local://tcm-platform/standard/assets/tongue_origin/ab/{asset_id}.jpg
local://tcm-platform/datasets/DS-PULSE-001/v2026.05.demo.001/manifest.jsonl
```

后续迁移到对象存储时可映射为：

```text
s3://tcm-platform/{relative_path}
```

业务代码只依赖 URI resolver，不直接拼接绝对路径。

## 6. asset_type 字典

首期建议固化以下类型：

| `asset_type` | 说明 | 默认训练可用 |
| --- | --- | --- |
| `source_excel` | 原始 Excel | 否 |
| `source_csv` | 原始 CSV | 否 |
| `source_json` | 原始 JSON | 否 |
| `pulse_raw_left` | 左手脉波原始文件 | 是 |
| `pulse_raw_right` | 右手脉波原始文件 | 是 |
| `pulse_image` | 脉图 | 视任务而定 |
| `tongue_origin` | 舌图原图 | 是 |
| `tongue_cut` | 舌图裁剪图 | 是 |
| `tongue_segmented` | 舌图分割图 | 是 |
| `tongue_region` | 舌图分区图 | 视任务而定 |
| `face_origin` | 面图原图 | 是 |
| `face_region` | 面部分区图 | 视任务而定 |
| `voice_zip` | 语音压缩包 | 否 |
| `voice_wav` | 语音 wav | 是 |
| `report_pdf` | PDF 报告 | 否 |
| `signature_image` | 签名图 | 否 |
| `recommendation_material` | 穴位/调理建议素材 | 否 |
| `preview_image` | 预览图 | 否 |
| `dataset_manifest` | 数据集 manifest | 否 |
| `benchmark_prediction` | benchmark 预测文件 | 否 |

## 7. 命名规则

### 7.1 batch_id

```text
{source_vendor}_{YYYYMMDD}_{sequence}
```

示例：

```text
zhongke_20260516_001
yushengtang_20260516_001
```

### 7.2 source_record_group_id

中科：

```text
zhongke_{source_visit_id}_{canonical_user_key_hash}_{cluster_start_yyyymmddhhmm}
```

玉生堂：

```text
yushengtang_{TreatNumber}
```

### 7.3 dataset version

```text
v{YYYY}.{MM}.{scope}.{sequence}
```

示例：

```text
v2026.05.demo.001
v2026.05.pulse.001
```

## 8. 与数据库字段映射

`fact_file_asset` 建议字段与本地存储对应关系：

| 字段 | 对应 |
| --- | --- |
| `asset_id` | 标准资产文件名主体 |
| `visit_id` | 所属 visit |
| `modality_record_id` | 所属模态 |
| `asset_type` | 标准资产类型 |
| `asset_role` | `raw/standard/preview/dataset/recommendation_material` |
| `file_name` | 原始文件名 |
| `file_path` | 原始绝对路径或审计路径 |
| `storage_uri` | `local://tcm-platform/...` |
| `file_hash` | `sha256:{hash}` |
| `file_size` | 字节数 |
| `mime_type` | MIME |
| `created_at_from_file` | 原始文件时间 |
| `parsed_success_flag` | 是否可解析 |

建议后续补充字段：

- `source_batch`
- `source_vendor`
- `source_visit_id`
- `source_record_group_id`
- `source_relative_path`
- `magic_bytes`
- `encoding`
- `asset_subtype`
- `side`
- `body_part`
- `is_training_eligible`
- `excluded_reason`

## 9. 一阶段落地建议

一阶段不需要真实大规模文件迁移，但建议先建立目录和 mock 文件：

```text
storage/
  raw/demo/demo_20260516_001/
    manifest.json
    source/
  standard/records/demo/
  standard/assets/pulse_raw_left/
  standard/assets/tongue_origin/
  datasets/DS-PULSE-001/v2026.05.demo.001/
    dataset_card.md
    manifest.jsonl
```

一阶段 Flask API 可先从 PostgreSQL 读取 demo 元数据；文件预览如暂时没有真实文件，可返回占位资产 URI，但 URI 形态必须符合本文规范。

## 10. 访问与权限

推荐权限：

- `raw/`：仅数据工程和管理员可读。
- `standard/assets/`：通过 Flask/Nginx 受控下载，不直接公开。
- `datasets/`：研究人员可下载脱敏版本。
- `labels/`：标注员和审核员按任务授权。
- `benchmarks/`：研究人员可读，提交者可写自己的 run。
- `quarantine/`：仅数据工程和管理员可读。

Nginx 不应暴露 `raw/` 目录。受控下载使用 Flask 鉴权后再通过 `X-Accel-Redirect` 或后端流式返回。

## 11. 清理策略

| 目录 | 清理策略 |
| --- | --- |
| `landing/incoming` | 已登记后可移动到 `raw`，空目录可清理 |
| `tmp` | 可定期清理 |
| `logs` | 按保留期清理或归档 |
| `standard/assets` | 不自动删除 |
| `raw` | 不自动删除 |
| `datasets` | 只归档，不覆盖 |
| `quarantine` | 人工处理后归档或转标准资产 |

## 12. 需要避免的做法

1. 不要让训练脚本直接扫描 `raw/`。
2. 不要用患者姓名作为标准资产路径。
3. 不要把 Windows 绝对路径作为唯一存储地址。
4. 不要覆盖同名批次目录。
5. 不要把穴位/建议素材混入舌图或面图训练目录。
6. 不要把 PDF、签名图默认放入研究导出。
