# 脉诊分析一阶段数据集构造与脚本

更新时间：2026-05-21

## 1. 低耦合原则

脉诊分析模块不直接扫描平台 `raw/` 原始目录，也不要求研究者连接平台数据库。平台侧先生成一个版本化研究数据集目录；脉诊分析脚本只消费该目录中的 manifest、表格和资产索引。

当前实现包含两个脚本：

| 脚本 | 职责 |
| --- | --- |
| `scripts/build_pulse_phase1_dataset.py` | 从平台 PostgreSQL 表导出脉诊一阶段研究数据集 |
| `scripts/analyze_pulse_phase1.py` | 从导出的数据集目录运行一阶段可信度分析 |

## 2. 数据集目录结构

默认输出路径：

```text
storage/datasets/DS-PULSE-PHASE1/v2026.05.phase1.001/
```

目录内容：

| 文件 | 说明 |
| --- | --- |
| `dataset_card.md` | 数据集说明 |
| `summary.json` | 样本量、导出时间、校验和、Parquet 写入状态 |
| `manifest.jsonl` | 样本级索引 |
| `measurements.csv` / `measurements.jsonl` | measurement 元数据 |
| `measurement_features.csv` / `measurement_features.jsonl` | measurement 级特征长表 |
| `waveform_manifest.jsonl` | 波形资产索引、预览点和摘要 |
| `position_features.csv` / `position_features.jsonl` | 部位级明细特征 |
| `feature_variables.csv` / `feature_variables.jsonl` | 变量字典 |
| `*.parquet` | 如果本地安装了可用 Parquet engine，则同步生成 |

Parquet 是优先格式，但不是一阶段硬依赖。没有 `pyarrow` 或 `fastparquet` 时，脚本仍会输出 CSV 和 JSONL。

## 3. 数据集构造方式

执行：

```bash
python scripts/build_pulse_phase1_dataset.py
```

可选参数：

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--dataset-id` | `DS-PULSE-PHASE1` | 数据集 ID |
| `--version` | `v2026.05.phase1.001` | 数据集版本 |
| `--output-root` | `storage/datasets` | 输出根目录 |
| `--output-dir` | 空 | 显式输出目录，设置后覆盖 `output-root/dataset-id/version` |

数据来源：

| 平台表 | 导出文件 |
| --- | --- |
| `fact_pulse_measurement` | `measurements.*` |
| `fact_pulse_measurement.feature_json` | `measurement_features.*` |
| `fact_pulse_waveform_asset` | `waveform_manifest.jsonl` |
| `fact_pulse_position_feature` | `position_features.*` |
| `dim_feature_variable` | `feature_variables.*` |

兼容策略：

- 如果一阶段新表已经存在，脚本优先从 `fact_pulse_measurement`、`fact_pulse_waveform_asset`、`fact_pulse_position_feature` 读取。
- 如果数据库尚未初始化 `sql/005_pulse_analysis_phase1.sql`，脚本会自动回退到当前 M2 已有结构：`fact_modality_record.parsed_structured_data_json.records[]`。
- 回退模式会从 records 中抽取数值字段作为 `measurement_features.*`，从 `waveform_preview` / `waveform_summary` 生成 `waveform_manifest.jsonl`，从中科 `measurements[]` 生成 `position_features.*`。
- 玉生堂 `dataPulse.json` 没有显式采样率字段；当前按设备默认 `500Hz` 写入 `sampling_rate`，并用完整波形 `sample_count / sampling_rate` 推断 `duration_seconds`。该推断规则会写入波形摘要中的 `sampling_rate` 和 `duration_seconds`。

`--parse-only` 入库行为：

- `python scripts/organize_offline_storage.py --parse-only` 会继续写入 `fact_modality_record.parsed_structured_data_json` 并重建 `mart_visit_feature_wide`。
- 如果已执行 `python -m backend.scripts.init_db` 并创建一阶段新表，`--parse-only` 还会同步填充 `fact_pulse_measurement`、`fact_pulse_waveform_asset`、`fact_pulse_position_feature` 和 `dim_feature_variable`。
- 如果一阶段新表不存在，接口响应中的 `pulse_analysis_sync_skipped` 会是 `true`，原有结构化解析不会因此失败。

该脚本是平台内部导出脚本，需要能访问项目后端依赖和 PostgreSQL。运行前应完成：

```bash
pip install -r requirements.txt
python -m backend.scripts.init_db
python -m backend.scripts.seed_demo_data
```

如果是在 WSL/Ubuntu 环境运行，应确认 `DATABASE_URL` 指向可访问的 PostgreSQL。

## 4. 一阶段分析方式

执行：

```bash
python scripts/analyze_pulse_phase1.py \
  --dataset-dir storage/datasets/DS-PULSE-PHASE1/v2026.05.phase1.001
```

默认输出：

```text
storage/datasets/DS-PULSE-PHASE1/v2026.05.phase1.001/analysis/phase1/
```

输出文件：

| 文件 | 说明 |
| --- | --- |
| `measurement_quality.csv` | 单次测量质量、漂移、可推断时长、推荐片段和有效性标签 |
| `channel_signal_quality.csv` | 通道级周期信号质量，包含寸/关/尺/overall 的 `periodic_snr`、`pulse_energy`、`template_coherence`、`residual_fluctuation` 和疑似未对准评分 |
| `measurement_channel_summary.csv` | 单次 measurement 的三通道汇总，包含寸/关/尺 SNR、能量、模板一致性、疑似未对准评分、有效通道数和三通道能量均衡指数 |
| `longitudinal_channel_summary.csv` | 用户-通道纵向汇总，包含同一患者同一通道的平均/最佳周期信噪比、能量、模板一致性和疑似未对准计数 |
| `feature_reliability.csv` | 特征缺失率、异常率、CV、ICC、设备/漂移敏感性和 A/B/C/D 分级 |
| `device_consistency.csv` | 近时间跨设备配对的偏差和合并策略 |
| `summary.json` | 分析摘要 |
| `report.md` | 简要报告 |
| `*.parquet` | 如果环境支持 Parquet，则同步输出 |

## 5. 当前算法边界

一阶段脚本先提供可运行的可信度分析骨架，重点是数据契约、可复现输出和质量字段形态。

当前实现：

1. 从 `waveform_manifest.jsonl` 的 `preview_json` 估计基线漂移、伪影比例和漂移严重度；兼容 `{name, points}` 和点列表两类预览结构。
2. 将 `channel_name` 标准化为 `cun`、`guan`、`chi`、`overall`，对每个通道的预览点做线性去趋势、残差波动和自相关峰值分析，输出 `periodic_snr`、`pulse_energy`、`template_coherence`、`residual_fluctuation` 和 `alignment_suspicion_score`。
3. 三通道质量判断优先于单纯稳定性：低幅、低周期性的稳定信号会被标记为 `low_snr` 或 `suspected_misalignment`，不会因为稳定而直接视作高质量脉波。
4. 输出 `valid`、`partial_valid`、`invalid` 三类 `measurement_validity_label`。当存在通道级分析结果时，`valid` 需要具备可推断时长、至少两个有效寸关尺通道，以及足够的通道周期信号质量；缺少完整三通道证据但有可用周期信号的样本标为 `partial_valid`。
5. 基于缺失率、异常率、变异系数、设备敏感性和漂移敏感性输出特征可靠性 A/B/C/D 分级。
6. 在同一用户近时间窗口内寻找跨设备配对，输出设备偏差和合并策略。

当前真实导出数据集 `DS-PULSE-PHASE1/v2026.05.phase1.001` 的一阶段分析结果已生成在：

```text
storage/datasets/DS-PULSE-PHASE1/v2026.05.phase1.001/analysis/phase1/
```

补充玉生堂 `duration_seconds` / `sampling_rate` 并加入通道级周期信号分析后，本次数据集中共有 804 条 measurement，其中 389 条为 `valid`，374 条为 `partial_valid`，41 条为 `invalid`。通道级输出包含 3131 条 waveform channel 记录，其中 3005 条为 `valid`，122 条为 `low_snr`，4 条为 `suspected_misalignment`。仍有 18 条 measurement 缺少可推断时长；中科 Excel 当前只包含统计特征，尚未接入原始脉搏波文件，因此不推断采样率和时长。跨设备一致性未产生结果，因为 30 分钟窗口内没有形成可用的跨设备同特征配对。

需要注意：当前 `periodic_snr`、`pulse_energy`、`template_coherence` 和 `alignment_suspicion_score` 仍基于 `waveform_manifest.jsonl` 中的 160 点预览波形计算，适合作为一阶段离线筛查和前端可视化输入；后续接入完整波形文件后，需要替换为滑动窗口、模板对齐和去趋势波动性分析的全量实现。

后续增强：

1. 接入完整波形文件，替换基于预览点的漂移估计。
2. 增加真实滑动窗口切分、峰谷检测和搏动级特征。
3. 增加补偿前后对比和推荐片段特征重算。
4. 将设备一致性从简单配对偏差升级为 Bland-Altman、ICC 和混合效应模型。
5. 增加图表输出。

## 6. 与通用下载能力的关系

当前 `build_pulse_phase1_dataset.py` 仍从平台数据库读取，属于平台内部导出脚本。未来平台通用下载接口完成后，应由通用 dataset builder 产出同等目录结构，脉诊分析脚本无需修改。

因此，稳定契约不是 `/api/pulse/*`，而是数据集目录中的：

- `manifest.jsonl`
- `measurements.*`
- `measurement_features.*`
- `waveform_manifest.jsonl`
- `position_features.*`
- `feature_variables.*`
