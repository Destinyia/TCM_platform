# 中医四诊数据平台 - 开发计划

## Context

医院和学校的科研工作需要整合来自多种四诊仪设备、HIS系统、科研数据库的异构数据。目前数据分散在不同平台，格式不统一，难以进行跨数据源的科研分析。本平台旨在建立统一的数据标准，实现多源数据的导入、标准化处理和按需导出，支撑中医药科研工作。

## 四、前端页面规划

### 4.1 数据模型说明

系统采用患者-检查记录的一对多关系，每个患者拥有多次检查记录，形成纵向追踪序列，支持观察患者病情随时间的演变趋势。

```
患者 (Patient)
  └── 检查记录1 (Examination) ─ 2024-01-15
  │     ├── 舌象数据
  │     ├── 面象数据
  │     ├── 脉诊数据
  │     ├── 中医辨证
  │     └── 生化指标
  └── 检查记录2 (Examination) ─ 2023-12-20
  │     └── ...
  └── 检查记录3 (Examination) ─ 2023-11-05
        └── ...
```

### 4.2 页面结构

```
frontend/
├── src/
│   ├── views/
│   │   ├── Layout.vue                   # 主布局框架

│   │   │

│   │   │   ├── 患者基本信息卡片

│   │   │       ├── 舌象模块
```

### 4.3 核心页面说明

| 页面 | 功能 | 说明 |
|------|------|------|
| Dashboard | 数据概览 | 统计卡片、数据源分布图、最近导入记录 |
| Patients | 患者列表 | 搜索、筛选、跳转追踪记录 |
| PatientDetail | 纵向追踪 | 时间线展示患者所有检查记录，支持病情演变分析 |
| Examinations | 检查记录 | 全局检查记录列表，支持按数据源/日期筛选 |
| Import | 数据导入 | 步骤向导：选择数据源→上传文件→字段映射→完成 |
| Export | 数据导出 | 选择格式、数据源、字段，生成标准化数据集 |

## 五、数据标准化方案

### 5.1 标准字段定义

建立中医四诊数据标准字典：

| 类型 | 特征字段 | 分类数 |
|------|----------|--------|
| 舌象 | 舌色、舌形、苔色、苔质 | 5/8/4/6类 |
| 面象 | 面色、光泽度、部位特征 | 6类 |
| 脉象 | 脉象类型、脉率、脉律 | 28种 |
| 闻诊 | 声音特征、气味特征 | - |

### 5.2 映射机制

```python
# 字段映射配置示例
YUSHENGTANG_TONGUE_MAPPING = {
    "source_field": "tongueColor",
    "target_field": "features.tongue_color",
    "transform": "value_map",
    "value_map": {
        "1": "淡白", "2": "淡红", "3": "红",
        "4": "绛红", "5": "紫暗"
    }
}
```
