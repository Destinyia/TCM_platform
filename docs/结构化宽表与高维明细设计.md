# 结构化宽表与高维明细设计

更新时间：2026-05-20

## 1. 目标

解析后的多源数据需要形成可直接用于展示、筛选、导出和机器学习 baseline 的结构化数据视图。第一阶段采用 visit 级宽表：

- 纵轴：一次标准化 `visit`。
- 横轴：从问诊、脉诊、舌诊、面诊、声诊等模态中抽取的变量。
- 追溯：保留原始资产和 `fact_modality_record.parsed_structured_data_json`，宽表只作为可重建的分析视图。

## 2. 当前落库结构

新增表：`mart_visit_feature_wide`

| 字段 | 说明 |
| --- | --- |
| `visit_id` | 主键，对应 `fact_visit.visit_id` |
| `user_id` | 用户 ID |
| `source_vendor` | 来源，当前为 `zhongke` / `yushengtang` |
| `source_visit_id` | 来源 visit ID |
| `source_record_group_id` | 标准化聚合记录 ID |
| `visit_time` / `visit_date` / `visit_slot` | 采集时间和早中晚槽位 |
| `quality_status` | visit 质量状态 |
| `modalities` | 当前 visit 存在的模态列表 |
| `feature_json` | 扁平化变量字典，可视为宽表横轴 |
| `feature_groups_json` | 按模态分组的变量，供详情页展开/折叠 |
| `feature_count` | 抽取出的变量数量 |
| `parser_version` | 宽表解析版本 |
| `updated_at` | 最近重建时间 |

第一阶段采用“固定元数据列 + JSONB 动态变量”的混合宽表。原因是当前玉生堂和中科字段仍在快速演进，直接把所有变量固化为 SQL 物理列会导致频繁迁移。后续变量字典稳定后，可将高频变量从 `feature_json` 固化为物理列。

## 3. 第一阶段变量抽取范围

### 问诊

玉生堂 `dataAsk.json`：

- `ask_question_count`
- `ask_q_{QuestionId}_answer`
- `ask_q_{QuestionId}_question`

中科问诊 Excel：

- 当前先保留解析资产和匹配行。
- 后续从 `问诊汇总`、体质辨识和五态人格 sheet 中抽取体质分值、主体质、倾向质、五态人格分值。

### 脉诊

玉生堂 `dataPulse.json`：

- `pulse_pulse_numbers`
- `pulse_amplitude`
- `pulse_speed`
- `pulse_strength`
- `pulse_peak_time_swell`
- `pulse_peak_time_sag`
- `pulse_yi_mai_percent`
- `pulse_ju_mai_percent`
- `pulse_wen_mai_dx_percent`
- `pulse_wen_mai_qr_percent`
- `pulse_wen_mai_km_percent`
- `pulse_ren_mai_percent`
- `pulse_start_time`

大段波形字段如 `CunShang`、`Cun`、`GuanMai`、`Chi`、`ChiXia`、`SinglePluse` 不直接写入宽表，只保留样本数、均值等摘要。原始波形后续进入专门资产或 Parquet/NPY 文件。

中科脉诊 Excel：

- 后续拆为 `fact_pulse_position_feature`。
- 宽表可同步保留六部位关键摘要，例如 `zk_pulse_left_cun_h1`、`zk_pulse_right_chi_w_t`。

### 舌诊

玉生堂 `dataTongue.json`：

- 舌色、舌体、厚薄、裂纹、胖瘦、歪斜、润燥、分析结论等稳定字段。
- 图片 URL 字段不进入宽表，图片资产仍由 `fact_file_asset` 管理。

中科舌诊 Excel：

- 后续拆为 `fact_tongue_region_color`，保存区域、色彩空间、通道和值。

### 面诊

中科面诊 Excel：

- 当前先保留解析资产和匹配行。
- 后续抽取面色、唇色、光泽、上中下庭、区域颜色指标。

### 声诊

玉生堂 `dataVoice.json`：

- `voice_qualitative`
- `voice_voice_name`
- `voice_kou`、`voice_yue`、`voice_dong`、`voice_ni`、`voice_hai`、`voice_yun`、`voice_xin`、`voice_shi`、`voice_fang`、`voice_mei`
- `AnalysisResults` 中每个音节的 `percentage`、`standard_value`、`energy`、`rms`、`zcr`、`pitch`

## 4. 超过二维的数据

以下数据不适合直接进入 visit 级宽表原始字段：

| 数据 | 维度 | 建议结构 |
| --- | --- | --- |
| 脉搏波形 | visit × channel × sample_index | `fact_pulse_waveform_asset` 或 Parquet/NPY 文件索引 |
| 中科六部位脉诊 | visit × pulse_position × feature | `fact_pulse_position_feature` |
| 舌诊区域颜色 | visit × tongue_region × color_space × channel | `fact_tongue_region_color` |
| 声诊分项 | visit × voice_item × acoustic_feature | `fact_voice_item_feature` |
| 问诊答案 | visit × question_id × answer | `fact_ask_answer` |

宽表只保存这些高维结构的关键摘要或 pivot 后常用变量。

## 5. 重建流程

当前重建入口：

```bash
python scripts/organize_offline_storage.py --parse-only
```

或直接调用：

```bash
curl -X POST http://localhost:5000/api/ingest/parse-structured-data \
  -H "Content-Type: application/json" \
  -d '{}'
```

执行结构化解析时会同步重建 `mart_visit_feature_wide`。如果只补没有解析过的数据：

```bash
python scripts/organize_offline_storage.py --parse-only --parse-only-missing
```

## 6. 前端展示

`GET /api/demo/visits/{visit_id}` 返回：

```json
{
  "feature_wide": {
    "feature_count": 120,
    "parser_version": "visit_feature_wide_v2",
    "updated_at": "2026-05-20T...",
    "features": {},
    "groups": {
      "base": {"label": "基础信息", "fields": {}},
      "ask": {"label": "问诊", "fields": {}},
      "pulse": {"label": "脉诊", "fields": {}}
```

visit 详情抽屉中新增“结构化宽表字段”区域。由于字段较多，前端按分组折叠展示，每个分组内部用紧凑字段网格显示变量和值。

## 8. 脉诊解析当前实现

当前脉诊解析采用“结构化 records + 高维明细摘要”的方式。

### 玉生堂 `dataPulse.json`

解析入口：`backend/app/pulse_parser.py`

标准化输出写入 `fact_modality_record.parsed_structured_data_json.records[]`，核心字段包括：

| 字段 | 说明 |
| --- | --- |
| `pulse_rate` | 脉率，对应 `PulseNumbers` |
| `force` | 脉力标准化指标，当前来自 `Strength` |
| `tension` | 紧张度指标，当前来自 `WenMai_QR_Percent` |
| `fluency` | 流利度指标，当前来自 `JuMai_Percent` |
| `amplitude` | 幅值，对应 `Amplitude` |
| `stability_score` | 有效脉段比例换算得分 |
| `waveform_summary` | 波形通道样本数、最小值、最大值、均值 |
| `waveform_preview` | 前端折线预览使用的降采样波形点 |

`waveform_preview` 由波形向量降采样生成，不来自 `pulseImage.jpg`。图片资产仍保留在 `fact_file_asset` 中用于原始资产查看。

### 中科 `脉诊数据*.xls`

解析入口同样为 `backend/app/pulse_parser.py`。

当前按 `visit.source_visit_id` 或 `cheat_types.raw_source_visit_id` 匹配 Excel 行块，输出：

- 整体脉象结果。
- `pulse_rate`、`force`、`tension`、`fluency`、`amplitude`、`stability_score` 聚合指标。
- `measurements[]` 六部位明细，包含部位、脉率值、脉力值、紧张度值、流利度值、`h1-h5`、`t/t1/t4/t5`、`W`、`As`、`Ad` 等。

这些六部位明细超过 visit 级二维宽表粒度，当前保存在 records 内；后续应迁移到 `fact_pulse_position_feature`。

### 前端展示约定

- 脉诊分析页读取 `GET /api/demo/pulse/records`，不在前端解析原始 JSON/XLS。
- 玉生堂记录展示波形向量降采样折线。
- 中科记录展示六部位测量折线和可展开明细表。
- visit 详情抽屉在脉诊模态下展示 records 表格，同时保留 assets 原始文件展示。
